


- titre : indexeur de recherche
- problématique : 'je veux cherche un fichier mais je ne sais pas ou '



- Structure
    /src
    |_indexes/ # contient les fichiers d'indice (chaque fichier contient un nom unique)
    |   |_ 0.json
    |   |_1.json 
    |_tools/ # contient les fichiers python utils
    |   |_ file.py # pour les opérations de fichier
    |   |_ find.py # pour tout ce qui est recherche
    |   |_ params.json # contient les parameters de l'app
    |   |_ params.py # load/update parameters as the user decide
    |_gui/ # contient les fichiers de l'interface utilisateur
        |_...
    |
    |_ new_files_indexes/ # contient les fichiers nouveaux qui ne sont pas encore indexés
        |_ 0_new.json 

- description de processus : 
    [!]  le fichier paramétre contient "SCAN_NEW_FILE" est un boolean , si  activé l'app scan pour tous les nouveaux fichiers qui apparaît dans les "PATH_TO_SCAN" et qui ne sont pas encore indexé
    [!]  le fichier paramétre contient "IGNORE_PATH" qui est une list des dossiers à ne pas inclure dans le scan (ne pas scanner l'interieur)
    [!]  le fichier paramétre contient "PATH_TO_SCAN" qui est une autre list qui contient les paths des dossiers à scanner seulement , en inclusant les childs paths de ces dossiers 
    [!]  le fichier paramétre contient "EXT_TO_IGNORE" qui est une autre list qui contient les extensions des fichiers à new pas scanner , exemple : [".py",".csv"]

- actions :  
    - uploader: 
        1.  l'utilisateur importe un fichier pour le sauvegarder dans l'application
        2. ajouter des mots clé ou une description ou bien des 'tags' à ce fichier uploadé
        3. l'application  utiliser ces mots clés , descriptions ... pour  créer son fichier d'index indexes/*.json
    - search : 
        1. l'utilisateur cherche dans la barre de recherche selon le type : 'par mot clé' , 'par mots dans description'
        2. si le fichier apparaît dans les résultats de recherche ,et l'utilsateur le  click  , l'app l'ouvrir dans l'explorateur
    
    - scan : 
        1. si "SCAN_NEW_FILE" dans le fichier parametre est activé , une fois l'app est lancé , il affiche 'scanning for new files'
        2. si il decouvre de nouveaux fichiers , il les ajoute dans la list , until the user decides to add tags or description to make an indexer file *.json for them
        [!] les fichiers scannés qui sont nouveaux , s'affichent sous forme de list-item , et si clické , l'app afficher formulaire , pour que l'utilisateur ajoute les infos pour créer un indexeur pour ce fichier
    + si un index pour un fichier contient le path d'un fichier qui est supprimé , on le update et mettre sont couleur en rouge
